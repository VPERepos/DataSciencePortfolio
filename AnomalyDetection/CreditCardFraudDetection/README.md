# Credit Card Fraud Detection.
## Problem Definition.
This project is based on a simulated dataset (https://www.kaggle.com/datasets/kartik2112/fraud-detection?select=fraudTrain.csv) consisting of transactions with variables describing merchants, card holders and transactions. Each row is labeled with fraud or not variable. The goal is to predict wether a transaction is a fraud. Problematic is the fact that frauds are extremly rare ($\le$ 1% ) and the classification becomes highly imbalanced.
## Technologies and libraries used.
For specific for the anaomaly detection problems feature engineering such a statistical approach like **Z-Score** calculation was conducted in order to describe a "normal" behaviour in transactions. Some categorical values like job description were highly nonunified, with duplicated job descriptions differing in small details from each other, so a process of recategorization had to be applied using the **Semantic Similarity Search** for finding according job cathegories in ISCO 8 standard. For calculation of anomaly scores such nonsupervised model like **Isolation Forest** was used. And for final classification the **XGBoost** model was chosen as the best performing. For the best model hyperparameters search a **Scikit Learn** function called **RandomisedSearchCV** was used.
